# R2-CLI 项目架构分析

## 项目概述

R2-CLI 是一个面向 AI 开发者的命令行工具，提供二手潮奢交易的全链路能力。项目基于 TypeScript 开发，使用 React 渲染交互界面。

## 项目结构

```
r2-cli/
├── src/
│   ├── commands/           # 命令模块
│   │   ├── auth/          # 认证相关命令
│   │   │   ├── login.ts   # 登录命令
│   │   │   └── executor.ts # 执行器
│   │   ├── auto-send/     # 自动发送模块
│   │   └── index.ts       # 命令入口
│   ├── query/             # 查询模块（已删除）
│   ├── services/          # 服务层
│   │   └── api/           # API 服务
│   │       └── api-client.service.ts  # API 客户端服务
│   ├── entrypoints/       # 入口点（新）
│   ├── main.tsx          # React 应用入口
│   └── index.ts          # CLI 入口
├── scripts/               # 构建脚本
│   ├── build.js          # 构建主脚本
│   ├── clean.js          # 清理脚本
│   ├── dev.js            # 开发脚本
│   └── fix-build.js      # 修复构建脚本
├── esbuild.config.js      # esbuild 配置
├── package.json          # 项目配置
└── dist/                 # 构建输出
```

## 核心架构组件

### 1. CLI 入口层

- **index.ts**: CLI 的主要入口点，使用 Commander.js 处理命令行参数
- **bin/r2**: 可执行文件入口，指向 dist/cli.js

### 2. 命令系统

采用模块化的命令设计，每个功能封装为独立的命令：

#### 认证模块 (auth/)
- **login.ts**: 处理用户登录逻辑
- **executor.ts**: 登录操作的执行器

#### 命令组织
- 命令通过 Commander.js 进行注册和管理
- 支持子命令结构（如 `auth login`）

### 3. 服务层

#### API 客户端服务 (services/api/)
- **api-client.service.ts**: 核心 API 客户端
  - 处理认证和请求头
  - 统一的错误处理
  - 支持 RESTful API 调用

### 4. 用户界面

#### React 应用 (main.tsx)
- 使用 Ink.js 在终端中渲染 React 组件
- 支持交互式界面和实时更新

#### UI 组件库
- 基于构建时的环境变量进行 UI 配置
- 支持二维码显示、表单交互等

## 构建系统

### 构建流程
1. **开发模式** (`scripts/dev.js`): 适合开发调试
2. **生产构建** (`scripts/build.js`): 生成可分发版本
3. **清理构建** (`scripts/clean.js`): 清理构建产物

### 构建工具
- **esbuild**: 高性能的 JavaScript 打包工具
- **TypeScript**: 类型安全的源代码
- **环境变量替换**: 构建时注入环境配置

## 数据流向

```
用户命令 → Commander.js → 命令模块 → API 服务 → AI 服务 → 响应处理 → UI 渲染
```

## 设计模式

### 1. 服务层模式
- API 调用封装在服务层
- 统一的错误处理和响应格式

### 2. 命令模式
- 每个命令都是独立的类/函数
- 通过 Commander.js 统一管理

### 3. 环境配置
- 构建时替换环境变量
- 支持不同环境的配置

## 关键特性

### 1. AI 能力集成
